## 1. 요약 (TL;DR)

- 실제 업체 구분은 **`tb_project.name`** 단위. `site_id`는 의미 없음(활동 사이트는 Site 1·2 두 곳뿐).
- Q&A 후보 1,358건이 약 200여 프로젝트에 분포 — **상위 25개 프로젝트가 약 50%** 차지.
- 도메인은 **LMS(학습관리시스템) 운영 지원이 압도적** — 교육원·대학교·기업교육 위주.
- 시점은 대부분 **2022년 상반기(3~6월)**, 극소수만 2026년. **답변 노후화 위험**.
- 빈출 주제: 일반 문의 > 오류 신고 > 진도/수강/학습 > 회원/계정 > 콘텐츠/영상 > SMS/메일 > 견적/정산 > 평가/시험 > 수료증 > 도메인 > 결제/PG.

## 2. 분류 단위 — 사이트 ≠ 업체

| 단위 | 의미 | Q&A 분포 |
| --- | --- | --- |
| `tb_site` | 시스템 인스턴스 | Site 1(맑은소프트) 1,146 / Site 2(맑은소프트 내부) 212. **나머지 61개 사이트는 활동 없음** |
| `tb_project` | 실제 업체·업무 단위 | 1,358 전부 프로젝트 매핑됨 |

**규칙**: 업체별 자료 격리·필터·접근 제어의 기본 키는 **`project_id`**. `site_id`는 외부 vs 내부 1차 분류용 정도.

## 7. AI 데이터 활용 시사점

| 시사점 | 적용 방안 |
| --- | --- |
| 인덱스 분리 키 = `project_id` | 업체별 자료 격리·필터·접근 제어의 기본 단위 |
| **사내 프로젝트 제외 룰** | 이름 패턴 `^[0-9]+\.` (예: `01. ...`, `09. ...`) + Site 2 일부 + `몽골개발팀` 등 화이트리스트 검증 후 제외 |
| **영업 일반LMS 별도 처리** | project_id=153. Q&A 형식 깨짐 → 챗봇 답변 자산보다는 **영업 사례 메타** 또는 **표준 답변 트리거** 검토 대상 |
| **표준 답변 자동 후보화** | 키워드 빈도(진도/수료/평가/SMS/도메인/PG 등)에서 반복 답변 다수 → 제목·본문 임베딩 클러스터링으로 표준 답변 후보 추출 |
| **업체별 답변 분리** | 같은 "수료 처리"라도 업체별 정책·세팅 상이 → 검색 시 `project_id` 필터 + 전사 공통 결과 병합 전략 |
| **답변 노후화 가중치** | 데이터 대부분 2022년 → `reg_date` 기반 신선도 가중치 + 정기 검토 워크플로 필수 |
| **종료 프로젝트(`*`) 가드** | 학습 자산으로는 유효하나 **"현재 시스템과 다를 수 있다"** 가드 필요 |
| **HRD 특화 카테고리** | 산인공 API, K-디지털, mOTP, WBT/PWBT, 나이스평가정보 등은 별도 표준 답변 카테고리로 구축 가치 |

## 8. 권장 후속 작업

1. **사내 프로젝트 필터 룰 SQL 작성**: `project.name REGEXP '^[0-9]+\\.'` + 수동 화이트리스트 검토 → CS 자산만 남김.
2. **Top 5 프로젝트 표본 검수** (에듀야·웅진·프라임에듀넷·케이에이치·경복대 각 30건) → 답변 품질·신선도 평가.
3. **주제 카테고리 클러스터링 PoC**: 제목 임베딩으로 자동 분류, 표준 답변 후보 추출.
4. **영업 일반LMS(project_id=153) 활용 방안 결정**: 챗봇 학습 자산 / 영업 메모 / 미사용 중 택일.
5. **종료 프로젝트(`*`) 노출 정책 합의**: Phase 2 챗봇에서 어디까지 답변 자산으로 쓸지.
6. **공공/HRD 도메인 별도 표준 답변 트랙** 운영 검토 (산인공·K-디지털 등 규제 변동성 큰 영역).
7. **2026년 활동 데이터 추출**: 1.7만 게시글 중 2026년 데이터만 별도 분석 — 현재 운영 흐름과 차이 검증.
